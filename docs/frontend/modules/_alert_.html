addErrorAlert

  • addErrorAlert(error: Error, tempAlert?: ChildNode | null, divString?: string): void
    • Defined in alert.ts:105

    Accepts a javascript error and tries to insert an appropriate alert into the alert-div. If the error is known, it will give the user a nice alert string which is probably more descriptive than the string in the error object. If the error body is a string, it puts the string in the alert. If it is a django error (see ErrorBody , it will output that error). Otherwise it outputs the HTTP status code error, or the javascript error that was encountered.

    Parameters

    • error: Error

      The error object

    • Default value tempAlert: ChildNode | null = null

      The temporary alert to remove on adding the new alert.

    • Default value divString: string = "alert-div"

      the div to add this to

    Returns void

    the alert as Node or null if it has already been deleted in some edge case
